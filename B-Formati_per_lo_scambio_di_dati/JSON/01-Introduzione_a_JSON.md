# 1. Introduzione a JSON

## Cos'è JSON (JavaScript Object Notation)

JSON (JavaScript Object Notation) è un formato di scambio dati leggero, facile da leggere e scrivere per gli esseri umani e semplice da analizzare e generare per le macchine.

## Storia e origine

- Creato da Douglas Crockford all'inizio degli anni 2000
- Derivato dalla sintassi letterale degli oggetti JavaScript
- Standardizzato come RFC 7159 e ECMA-404

## Vantaggi e utilizzi principali

- **Leggibilità**: formato testuale facilmente comprensibile
- **Leggerezza**: struttura minimale senza tag ridondanti
- **Indipendenza dal linguaggio**: supportato dalla maggior parte dei linguaggi di programmazione
- **Utilizzi comuni**:
  - Comunicazione client-server (API REST)
  - File di configurazione
  - Archiviazione dati
  - Scambio dati tra applicazioni

## JSON vs XML vs altri formati

### JSON vs XML
- **JSON**: più compatto, meno verboso, nativo in JavaScript
- **XML**: più flessibile per documenti complessi, supporta attributi e namespace

### JSON vs CSV
- **JSON**: supporta strutture gerarchiche e tipi di dati complessi
- **CSV**: più semplice per dati tabulari piatti

### JSON vs YAML
- **JSON**: più rigoroso, meno propenso agli errori
- **YAML**: più leggibile, supporta commenti
